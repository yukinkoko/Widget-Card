# ウィジェット単語アプリ ― 決定事項まとめ（共有用）

最終更新: 2026-06-19 / リポジトリ: Widget-Card（Kotlin Multiplatform）

> 企画ブリーフ＋設計セッションで決まった内容を共有用にまとめたもの。元ブリーフ類は末尾「参考資料」を参照。
> モックアップ画像は `docs/mockups/` を参照。

## 1. コンセプト
- 一言: **覚えたい言葉を、毎日の画面に。**
- 補足: 単語・読み方・意味がホーム/ロック画面に常に表示され、**タップしなくても、ながら見で少しずつ覚えられる**。
- 「単語帳/テスト/クイズ」アプリではない。**毎日の画面に置いておく小さな言葉の学習アプリ**。

## 2. 対象ユーザー
- ライト学習層（英会話・韓国語などをなんとなく）
- 旅行前の実用層（旅行フレーズ）
- ながら勉強層（資格・仕事用語）
- 共通: 勉強したい気持ちはあるが毎日アプリを開く習慣はない。通知で急かされるより自然に目に入る方が続く。

## 3. 差別化
- ウィジェ単（クイズ型）との差: **常に表示・ながら見**。
- Lingo Widget（英語中心）との差: **英語に限定しない／目的別セットが主役**。
- AIフラッシュカード系との差: Phase 2 で**目的を伝えると言葉セットを作る**方向。

## 4. 技術構成・対象プラットフォーム
- **iPhone 優先**（ロック画面ウィジェットは事実上 iOS のみ）。
- **Kotlin Multiplatform テンプレートを活用**:
  - 本体アプリ = **Compose Multiplatform**
  - **ウィジェット = ネイティブ SwiftUI / WidgetKit**（Compose では作れない）
  - 本体 ↔ ウィジェットのデータ共有 = **App Group**
- ビルド/実行は Xcode。

## 5. デザイン言語
- **完全モノクロ**（白・薄グレーの面、墨の文字）。差し色なし。操作・アクティブ状態は**濃いインク**で表現。
- 書体: **サンセリフ統一**、ウェイトで階層。読み方は中間グレー。
- **静か・大人っぽい・日常に馴染む／毎日見ても疲れない**。学生・受験・ゲーム・カラフル・バッジ/ランキングは避ける。ただし**「ちゃんとしたアプリ」の作り込み（カード・タブ・進捗・明快な操作）は残す**。
- **色はトークン化**: 意味トークン `accent` を1つ用意し、今は**インク**に割当。**Phase 2 でこの1トークンを差し替えるだけ**で色味変更可（ユーザーの“きせかえ”にも拡張可）。Compose `ColorScheme` / SwiftUI `tint` / Figma Variables で共有。
- 角丸は控えめ、区切りはヘアライン、余白広め。

## 6. インタラクションモデル（コア）
- **受け身の表示回数メーター**: 単語に出会う（画面に出る）ほど**信号風の段階バー**が育つ（**タップ不要**）。
  - 段階: **1本 → 2本 → 3本**。**3本＝定着 → お休み（Resting）**。
  - サーフェス（ウィジェット/リスト）は段階バーのみ。**正確な回数は単語詳細だけ**に表示。
- **満タンで自動お休み** → 出番が減り、**たまに Review で再登場**。
- 自己採点・操作の義務はゼロ。**任意で**単語詳細に「**もう知ってる → 即お休み**」だけ用意。
- しきい値（何回で休む・復習間隔）は調整可（TBD）。

## 7. 用語・ラベル
- **UI（操作・状態）は英語**: `Today` / `Sets` / `Settings` / `Resting` / `Review` など。
- **学習コンテンツ（意味・例文）は日本語**。
- 単語・読み方は対象言語（例: 韓国語＋カナ読み）。

## 8. ウィジェット（3サイズ）
すべてモノクロ・段階バー付き・操作は最小。
- **小**: 単語 / 読み方 / 短い意味 / バー（＋音声ボタン）
- **中**: 単語 / 読み方 / 意味 / 例文1行 / バー（＋音声ボタン）
- **大**: 今日の3単語（各: 単語 / 読み方 / 短い意味 / バー）。お休みは淡色。
- **音声ボタン（P1 目標）**: 各単語にスピーカー（詳細は §10）。
- 図: `mockups/widgets.svg`, `mockups/meter-model.svg`

## 9. 画面構成
- **Home（Today）**: 今日の単語、進捗（Learned / Learning）、次に出る単語。各単語に段階バー。→ `mockups/home.svg`
- **Sets**: 目的別セットが主役。「使用中（ウィジェット表示中）」と「ライブラリ（＋追加）」を分離。各セットに進捗。手動作成は控えめ、AI生成は Phase 2 のプレースホルダ。→ `mockups/sets.svg`
- **単語詳細**: 大きな単語・読み方・意味・例文、表示回数（正確な回数）、音声ボタン、任意の「もう知ってる→お休み」。→ `mockups/word-detail.svg`
- **Onboarding（3ステップ）**: ①目的を選ぶ ②最初のセットを選ぶ ③**ウィジェット追加の案内**（肝）。→ `mockups/onboarding.svg`
- **Settings**（予定）: 1日に出す単語数 / ウィジェットの見た目 / 表示するセット / プライバシーポリシー。

## 10. 音声（P1 でウィジェット音声ボタン）
- 方針: **P1 からウィジェット上に音声（発音）ボタン**を入れる。
- **ホーム画面**: 実現可能（実例あり: WidgetClub 等）。仕組み = ボタン → **App Intent** → `AVAudioSession(.playback)` → `AVAudioPlayer` で**短尺バンドル音声**を再生。`.playback` はマナーモードでも鳴る。
- **ロック画面**: 制約が強く、**「アプリを開いて再生」フォールバック**の可能性 → 要確認。
- **進め方**: 実装初期に**「音声スパイク」**を1本作り、ホーム画面で安定再生＆ロック画面の可否を検証してから本実装。
- **音声アセット**: プリセットは固定語彙なので**発音を事前生成してバンドル**（P2 で録音/AI生成に拡張）。
- 影響: ウィジェットは表示専用ではなくなり、**App Intents が P1 に必要**。

## 11. スコープ
**Phase 1（公開初期）**
- 目的別プリセット単語セット（3〜5個）
- 今日の単語表示 / ホーム＆ロック画面ウィジェット
- 受け身の表示回数メーター → 自動お休み / Review
- 単語詳細 / 進捗の簡易表示 / セット選択
- ウィジェット音声ボタン（要検証）
- 手動の追加・編集は残す（主役にしない）

**Phase 2 以降**
- AI による目的別セット自動生成（確認画面必須）
- 音声の拡張（録音/AI、ウィジェット音声の本格化）
- 色テーマ（`accent` トークン差し替え／きせかえ）

**対象外（初期）**
- ランキング / 友達共有 / 学習時間記録 / 大量テストモード / 複雑な忘却曲線設定 / ノート・PDF・画像からの生成
