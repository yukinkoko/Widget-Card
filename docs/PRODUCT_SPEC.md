# ウィジェット単語アプリ ― 統合仕様（唯一の正 / Single Source of Truth）

最終更新: 2026-06-19 / リポジトリ: Widget-Card（Kotlin Multiplatform）

> 2つの並行セッション（本デザインセッション＋別の構成検討セッション）の決定を統合した版。
> 以前の `WIDGET_WORD_APP_SHARED_DECISIONS.md` / `WIDGET_WORD_APP_PHASE1_STRUCTURE.md` の内容はここに統合済み（一部は下記「統合メモ」で置き換え）。モックは `docs/mockups/`。

## 0. 統合で確定した3つの分岐（2026-06-19）
1. **主役 = 自作フォルダ＋手動登録**（プリセットはサンプルフォルダに降格）
2. **学習モデル = 受け身の表示回数メーターのみ**（期限スケジューリング・手動の「覚えた/まだ」は不採用）
3. **音声 = Phase 1 でウィジェット音声ボタン**（要・実装初期スパイク検証）

## 1. コンセプト
- 一言: **覚えたい言葉を、毎日の画面に。**
- ホーム/ロック画面に単語・読み方・意味が常に表示され、**タップしなくても、ながら見で少しずつ覚えられる**。
- 「単語帳/テスト/クイズ」アプリではない。**毎日の画面に置いておく小さな言葉の学習アプリ**。

## 2. ポジショニング（主役）
- **主役 = ユーザーが目的ごとに作る「フォルダ」＋手動の単語登録**。
- プリセットは廃止し、**サンプルフォルダ**として同梱（使い方を見せる補助）。空状態を避けるため最初から1つ入れる。
- 「単語帳1」のような無機質な名前は避け、目的が伝わる名前にする（例: 韓国旅行で使う単語 / 英語会議 / デザイナー英語 / カフェで使う言葉）。

## 3. 対象ユーザー
- ライト学習層（英会話・韓国語などをゆるく）
- **旅行前の実用層（Phase 1 の一番の狙い）**
- ながら勉強層（資格・仕事・制作で使う言葉）
- 共通: 勉強したい気持ちはあるが毎日アプリを開く習慣はない。通知で急かされるより自然に目に入る方が続く。

## 4. 差別化
- 常に表示・ながら見（クイズ型を主役にしない）
- 英語に限定しない／**目的別フォルダを自分で作れる**
- 静か・ニュートラル（学習アプリっぽさを抑える）。通知・ランキング・連続記録で煽らない
- AIフラッシュカード系との差: Phase 2 で「目的を伝えるとフォルダ＋単語を作る」方向

## 5. 技術構成・対象プラットフォーム
- **iPhone 優先**（ロック画面ウィジェットは事実上 iOS のみ）。
- **Kotlin Multiplatform テンプレートを活用**:
  - 本体アプリ = **Compose Multiplatform**
  - **ウィジェット = ネイティブ SwiftUI / WidgetKit**（Compose では作れない）
  - 本体 ↔ ウィジェットのデータ共有 = **App Group**
- ビルド/実行は Xcode。

## 6. デザイン言語
- **完全モノクロ**（白・薄グレーの面、墨の文字）。差し色なし。操作・アクティブ状態は**濃いインク**で表現。
- 書体: **サンセリフ統一**、ウェイトで階層。読み方は中間グレー。
- 静か・大人っぽい・毎日見ても疲れない。学生/受験/ゲーム/カラフル/バッジ・ランキングは避ける。ただし**「ちゃんとしたアプリ」の作り込み（カード・タブ・進捗・明快な操作）は残す**。
- **色はトークン化**: 意味トークン `accent` を1つ用意し、今は**インク**に割当。**Phase 2 で差し替えるだけ**で色味変更可（“きせかえ”にも拡張可）。Compose `ColorScheme` / SwiftUI `tint` / Figma Variables で共有。
- フォルダの識別は当面**モノクロのアイコン/濃淡**で（フォルダ色は Phase 2 のテーマ機能で）。

## 7. 学習モデル（コア）= 受け身の表示回数メーター
- **表示回数メーター**: 単語が画面に出る（出会う）たびに**信号風の段階バー**が育つ。**タップ不要**。
  - 段階: **1本 → 2本 → 3本**。**3本＝定着 → お休み（Resting）**。
  - サーフェス（ウィジェット/リスト）は段階バーのみ。**正確な回数は単語詳細だけ**に表示。
- **満タンで自動お休み** → 出番が減り、**たまに Review で再登場**。
- **期限スケジューリングは不採用**（別セッション案を統合時に見送り）。頻度は表示回数のみで自動調整。
- **手動の「覚えた/まだ」状態は持たない**。任意の利便機能として、単語詳細にだけ「もう知ってる → 即お休み」を置ける（最小・省略可）。
- しきい値（何回で Resting / Review 間隔）は調整可（TBD）。

## 8. 用語・ラベル
- 学習コンテンツ（意味・例文）は**日本語**。単語・読み方は対象言語（例: 韓国語＋カナ）。
- 状態語は基本不要（メーターのみ）。UI ナビ語の言語（英語: Today/Folders/Settings ↔ 日本語: ホーム/フォルダ/設定）は**未決**（§13）。

## 9. ウィジェット（3サイズ＋ロック画面）
すべてモノクロ・段階バー付き・操作は最小。
- **小**: 単語 / 読み方 / 短い意味 / バー（＋音声）。タップで詳細。
- **中**: 単語 / 読み方 / 意味 / 例文1行 / バー（＋音声）。
- **大**: 今日の3単語（各: 単語 / 読み方 / 短い意味 / バー）。お休みは淡色。
- **ロック画面**: 単語 / 読み方 / 短い意味（接触回数を増やす）。
- **音声ボタン（P1）**: §10。
- 図: `mockups/widgets.svg`, `mockups/meter-model.svg`

## 10. 音声（P1 でウィジェット音声）
- **P1 からウィジェット上に音声（発音）ボタン**を入れる。
- **ホーム画面**: 実現可能（実例あり: WidgetClub 等）。ボタン → **App Intent** → `AVAudioSession(.playback)` → `AVAudioPlayer` で**短尺バンドル音声**を再生（`.playback` はマナーモードでも鳴る）。
- **ロック画面**: 制約が強く、**「アプリを開いて再生」フォールバック**の可能性 → 要確認。
- **進め方**: 実装初期に**音声スパイク**を1本作り、ホーム画面で安定再生＆ロック画面の可否を検証 → 本実装。
- **音声アセット**: 発音を事前生成して**バンドル**（P2 で録音/AI生成に拡張）。
- 影響: ウィジェットは表示専用ではなくなり、**App Intents が P1 に必要**。

## 11. 画面構成
| 画面 | 役割 | タブ |
|---|---|---|
| ホーム（Today） | 今日の単語、選択中フォルダ、簡易進捗、各単語に段階バー | タブ1 |
| フォルダ一覧 | 作成したフォルダの管理・切替・新規作成 | タブ2 |
| フォルダ作成 | 名前（＋任意で説明/アイコン）を決める | （フォルダから遷移） |
| 単語登録 | 単語・読み方・意味・例文を手動登録（まず3語からの軽さ） | （フォルダから遷移） |
| 単語詳細 | 例文・使う場面・表示回数（正確）・音声・任意の「もう知ってる」 | （各単語から遷移） |
| ウィジェット案内 | ホーム/ロック画面への追加方法 | （オンボーディング/設定から） |
| 設定 | 表示数・更新タイミング・テーマ等（最小限） | タブ3 |

タブは **ホーム / フォルダ / 設定** の3つに抑える。

### オンボーディング（聞きすぎない）
1. コンセプトを短く見せる
2. **最初のフォルダを作る** or **サンプル（韓国旅行）を試す**
3. （自作なら）まず3語だけ登録 / サンプルなら登録を飛ばす
4. 今日の単語を見せる
5. **ウィジェット追加を案内**（肝）
6. ホームへ

初回で聞かない: 学習目標 / 1日の学習時間 / レベル診断 / 通知許可 / アカウント登録。

## 12. データモデル（統合・簡素化版）
### フォルダ
| 項目 | 必須 | 用途 |
|---|---|---|
| フォルダ名 | 必須 | 例: 韓国旅行で使う単語 |
| 説明 | 任意 | 何のためのフォルダか |
| アイコン/識別 | 任意 | モノクロで見分け（色は P2） |
| 単語一覧 | 必須 | 含まれる単語 |

### 単語
| 項目 | 必須 | 用途 |
|---|---|---|
| 単語 | 必須 | メイン表示 |
| 読み方 | 必須 | 発音補助 |
| 意味 | 必須 | ウィジェット表示 |
| 例文 | 任意 | 中サイズ/詳細 |
| 使う場面 | 任意 | 旅行・仕事などの文脈 |
| メモ | 任意 | 自由登録向け |
| 表示回数 | 必須 | メーターの素（何回出たか） |
| 最終表示日 | 必須 | 次に出すタイミング判断 |
| お休み状態 | 自動 | 表示回数から導出（手動ではない） |

> 別セッション案にあった「期限/目標日/推奨単語数」「手動の状態(覚えた/まだ)」「最終回答日」は、メーター方式採用に伴い**不採用**。

## 13. スコープ
**Phase 1（公開初期）**
- 自由なフォルダ作成＋手動の単語登録（主役）
- サンプルフォルダ同梱（韓国旅行）
- 今日の単語表示 / ホーム＆ロック画面ウィジェット
- 受け身の表示回数メーター → 自動お休み / Review
- 単語詳細 / 簡易進捗 / フォルダ切替
- **ウィジェット音声ボタン**（要検証）

**Phase 2 以降**
- AI による目的別フォルダ＋単語の自動生成（確認画面必須）
- 音声の拡張（録音/AI）
- 色テーマ（`accent` 差し替え／きせかえ／フォルダ色）

**対象外（初期）**
- AI自動生成 / 音声以外の拡張 / ランキング / 友達共有 / 学習時間記録 / 大量テストモード / 複雑な復習間隔の手動設定 / 期限スケジューリング / PDF・画像・ノートからの生成 / アカウント必須化

1. 覚えたい言葉を、毎日の画面に。
2. 単語・読み方・意味がいつも見える。
3. 旅行や仕事に合わせて**フォルダを作れる**。
4. ウィジェットで、アプリを開かずに復習できる。
5. 学習アプリっぽすぎない静かな見た目。
