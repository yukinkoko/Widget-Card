# ウィジェット単語アプリ Phase 1 構成案

> ⚠️ この資料は `PRODUCT_SPEC.md`（統合版・唯一の正）に統合されました。確定方針はそちらを参照してください。
> 統合時の変更: 期限スケジューリングと手動「覚えた/まだ」は不採用（受け身の表示回数メーターに一本化）、音声は Phase 1 に前倒し。フォルダ主役・手動登録・画面/データモデル構造は採用。

## 1. このアプリの芯

### コンセプト

覚えたい言葉を、毎日の画面に置いておくアプリ。

ユーザーがアプリを開いて勉強するのではなく、ホーム画面やロック画面を見るたびに、単語・読み方・意味が自然に目に入る状態をつくる。

### 一言でいうと

タップしなくても少しずつ覚えられる、ながら見の単語ウィジェット。

### 大事にすること

- 単語・読み方・意味が最初から見えている
- 勉強を急かさない
- 通知やランキングで煽らない
- 英語だけに限定しない
- ユーザー自身の目的ごとにフォルダを作れる
- 例フォルダで使い方をすぐ理解できる
- ウィジェットを置いた瞬間から価値が出る

### やらないこと

- 回答を隠すクイズ型を主役にしない
- テストモードを複雑にしない
- 学習時間や連続記録でプレッシャーをかけない
- 入力作業ばかりの単語帳アプリに寄せすぎない
- Phase 1ではAI生成や音声を入れない

## 2. Phase 1 のゴール

### ユーザーに届けたい状態

ユーザーが覚えたい言葉をフォルダに登録し、ホーム画面またはロック画面にウィジェットを置く。

その後は、アプリを開かなくても毎日数語が目に入り、覚えた / まだ の軽い記録だけで復習が回る。

### Phase 1 の成功条件

- 初回起動から3分以内にウィジェットを置ける
- ウィジェットだけで単語・読み方・意味がわかる
- 覚えていない単語が後日また出る
- いつまでに覚えたいかを指定できる
- 指定した期限に合わせて、出る回数や頻度が自然に変わる
- 自由にフォルダを作って単語を登録できる
- 例フォルダだけでも使い方がわかる

## 3. 想定ユーザー

### 優先ユーザー

1. 旅行前に少し言葉を覚えたい人
2. 英語や韓国語をゆるく続けたい人
3. 仕事や制作で使う言葉を日常的に見ておきたい人

### Phase 1 で一番狙うユーザー

旅行前の実用層。

理由:

- 目的がはっきりしている
- 使う場面を想像しやすい
- フォルダ分けとの相性が良い
- App Store のスクリーンショットで価値が伝わりやすい

## 4. アプリ全体の構成

### 画面一覧

| 画面 | 役割 | Phase 1 必須 |
|---|---|---|
| ホーム | 今日の単語、選択中フォルダ、進捗を見せる | 必須 |
| フォルダ一覧 | 作成したフォルダを管理する | 必須 |
| フォルダ作成 | 名前、期限、登録数の目安を決める | 必須 |
| 単語登録 | 単語、読み方、意味、例文を登録する | 必須 |
| 単語詳細 | 例文、使う場面、状態を確認する | 必須 |
| ウィジェット案内 | ウィジェット追加方法を案内する | 必須 |
| 設定 | 表示数、更新タイミング、テーマなど | 最小限 |

### タブ構成案

Phase 1 はタブを増やしすぎない。

推奨:

1. ホーム
2. フォルダ
3. 設定

単語詳細や単語登録は、タブではなく各フォルダから遷移する。

## 5. 初回体験

### 初回起動の流れ

1. コンセプトを短く見せる
2. 最初のフォルダを作る、または例フォルダを選ぶ
3. いつまでに覚えたいかを選ばせる
4. まず数語だけ登録する、または例フォルダで試す
5. 今日の単語を見せる
6. ウィジェット追加を案内する
7. ホーム画面へ戻る

### 初回で聞きすぎないこと

- 学習目標
- 1日の学習時間
- レベル診断
- 通知許可
- アカウント登録

初回は「フォルダを作る / 例フォルダで試す」と「ウィジェットを置く」だけに寄せる。

### 期限設定の見せ方

「記憶曲線」や「復習間隔」という言葉は出さず、次のように簡単に聞く。

- 1週間後まで
- 2週間後まで
- 1か月後まで
- 期限なしでゆっくり

旅行系フォルダの場合は「出発日」を入れられるとわかりやすい。

Phase 1 では、期限は単語ごとではなくフォルダ単位で持つのがおすすめ。

例:

- 韓国旅行で使う単語フォルダ: 7月20日までに覚えたい
- 英語会議フォルダ: 1か月かけて覚えたい
- デザイナー英語フォルダ: 期限なしでゆっくり

期限が近い場合は、おすすめ単語数を絞って伝えるほうがよい。

例:

| 期限 | おすすめ単語数 | 見せ方 |
|---|---:|---|
| 3日後まで | 5-10語 | まずは本当に使う言葉だけ |
| 1週間後まで | 10-20語 | 旅行前に覚える短期フォルダ |
| 2週間後まで | 20-30語 | よく使う言葉を広めに |
| 1か月後まで | 30-50語 | 少しずつ覚える標準フォルダ |
| 期限なし | 上限なし | ゆっくり増やせる |

強く制限するより、「この期限なら10語くらいがおすすめです」のように提案する。

ただし、短期フォルダでは多すぎる登録に軽い警告を出す。

### フォルダ作成フロー

フォルダ作成は、最初から細かく設定させすぎない。

Phase 1 では次の流れがおすすめ。

1. フォルダ名を入れる
2. いつまでに覚えたいかを選ぶ
3. おすすめ単語数を見る
4. 最初の3語を登録する、または例フォルダを試す
5. ウィジェットの見え方を確認する

画面上の聞き方は、専門用語を使わずにする。

例:

- いつまでに覚えたいですか？
- この期限なら、まず10語くらいがおすすめです
- 期限が近いので、よく使う言葉から登録しましょう
- あとから単語はいつでも追加できます

### フォルダ作成時の入力項目

| 項目 | 扱い | 見せ方 |
|---|---|---|
| フォルダ名 | 必須 | 例: 韓国旅行で使う単語 |
| 期限 | 必須 | 具体日 / 期間 / 期限なし から選ぶ |
| 説明 | 任意 | 何のためのフォルダかをメモできる |
| アイコン / 色 | 任意 | フォルダを見分けやすくする |
| 表示ペース | 詳細設定 | 基本は自動。必要な人だけ変更できる |

最初の作成画面では、必須項目だけを目立たせる。

説明、アイコン、色、表示ペースは後から変更できる設定にする。

### 期限の入力方法

期限は、次の3つから選べる形がわかりやすい。

| 入力方法 | 向いている使い方 | 表示例 |
|---|---|---|
| 具体日 | 旅行、試験、発表など日付が決まっている | 7月20日まで |
| 期間 | ゆるい学習、仕事用語、趣味 | 1週間後まで / 1か月後まで |
| 期限なし | いつか覚えたい言葉を貯める | 期限なしでゆっくり |

旅行系フォルダでは、具体日を「出発日」として見せると自然。

例:

- 出発日までに覚える
- 発表日までに覚える
- 期限なしで少しずつ覚える

### 単語数のすすめ方

期限に対して単語数が多すぎる場合でも、基本は登録を止めない。

ただし、Phase 1 では「おすすめ数」をはっきり見せる。

例:

- 1週間後までなら、まず10-20語がおすすめです
- 30語あります。全部出すより、よく使う言葉から優先します
- 期限が近いので、重要な単語を先に表示します

登録を完全に止めるより、軽い注意と優先表示にするほうが自由度を保てる。

### 初回のおすすめ導線

初回は、次のどちらかを選べるようにする。

1. 自分でフォルダを作る
2. 「韓国旅行で使う単語」フォルダで試す

自分で作る場合は、まず3語だけ登録させる。

例フォルダで試す場合は、登録作業を飛ばして今日の単語とウィジェット案内へ進める。

## 6. ホーム画面構成

### 表示する内容

1. 今日の単語
2. 選択中のフォルダ
3. 覚えた / まだ
4. ウィジェットの見え方
5. 簡易進捗

### ホームでやること

- 今日の単語を確認する
- 覚えた / まだ をつける
- 詳細を見る
- フォルダを切り替える
- 単語を追加する
- ウィジェット追加方法を見る

### ホームでやらないこと

- 大量の単語一覧を見せない
- 学習時間グラフを大きく出さない
- クイズ開始ボタンを主役にしない
- AI生成導線を主役にしない

## 7. ウィジェット構成

### 共通方針

単語・読み方・意味は常に見えている状態にする。
タップして答えを表示するクイズ型にはしない。

### 小サイズ

目的:

一瞬で1語だけ目に入る。

表示:

- 単語
- 読み方
- 短い意味

操作:

- タップで単語詳細を開く

### 中サイズ

目的:

1語を少し深く見られる。

表示:

- 単語
- 読み方
- 意味
- 例文1行
- 覚えた / まだ

操作:

- 覚えた
- まだ
- 詳しく見る

### 大サイズ

目的:

今日の3単語をまとめて見る。

表示:

- 今日の3単語
- それぞれの読み方
- それぞれの短い意味
- それぞれの状態

操作:

- 各単語の詳細
- 可能なら覚えた / まだ

### ロック画面

目的:

さらに軽く、見るだけの接触回数を増やす。

表示:

- 単語
- 読み方
- 短い意味

操作:

- タップで詳細へ

## 8. 単語データの構成

### 1フォルダが持つ情報

| 項目 | 必須 | 用途 |
|---|---|---|
| フォルダ名 | 必須 | 例: 韓国旅行で使う単語 |
| 説明 | 任意 | 何のための言葉か |
| 目標日 | 任意 | 具体日を選んだ場合に使う |
| 期限タイプ | 必須 | 具体日 / 期間 / 期限なし |
| 表示ペース | 任意 | 基本は自動。詳細設定で変更できる |
| 推奨単語数 | 必須 | 期限に対して無理のない語数を自動で出す |
| 登録上限 | 任意 | 基本は止めず、短期フォルダで軽く注意する |
| 単語一覧 | 必須 | フォルダに含まれる単語 |

### 1単語が持つ情報

| 項目 | 必須 | 用途 |
|---|---|---|
| 単語 | 必須 | メイン表示 |
| 読み方 | 必須 | 発音の補助 |
| 意味 | 必須 | ウィジェット表示 |
| 例文 | 必須 | 中サイズ、詳細 |
| 使う場面 | 必須 | 旅行・仕事などの文脈 |
| カテゴリ | 任意 | フォルダ内の整理 |
| 難易度 | 任意 | Phase 2 以降でもよい |
| メモ | 任意 | 自由登録向け |
| 状態 | 必須 | 覚えた / まだ |
| 表示回数 | 必須 | 何回ウィジェットに出たか |
| 最終表示日 | 必須 | 次に出すタイミングの判断 |
| 最終回答日 | 必須 | 覚えた / まだ を押した日 |

### 状態

- 未確認
- まだ
- 覚えた

Phase 1 では、この3つで十分。

## 9. 復習ロジック

### 基本方針

複雑な設定画面は入れない。
ただし、記憶に残りやすい間隔で同じ単語が何度か出るようにする。

### Phase 1 のおすすめルール

- 覚えた: 少し間隔を空けてまた出す
- まだ: 近いうちにまた出す
- 未確認: 通常の候補として出す
- 期限が近い単語: 出る頻度を少し上げる
- 表示回数が少ない単語: 優先して出す
- 期限に対して単語数が多すぎるフォルダ: 重要度の高い単語から出す

### 期限に合わせた出し方

ユーザーが「いつまでに覚えたいか」を指定すると、アプリ側で出す回数を調整する。

例:

| 期限 | 出し方 |
|---|---|
| 1週間後まで | 毎日同じ単語が複数回出てもよい |
| 2週間後まで | まだの単語を短い間隔で出す |
| 1か月後まで | 覚えた単語も数日おきに再表示する |
| 期限なし | ゆっくり全体を回す |

旅行系フォルダでは、出発日から逆算して「よく使う単語」を優先する。

単語数が多すぎる場合は、全部を均等に出さない。

期限までに触れられる回数が少ないため、アプリ側で「まず覚える単語」を優先する。

### 表示の考え方

今日の単語は、選択中フォルダから3語を出す。

優先順:

1. 期限が近い
2. まだ
3. 表示回数が少ない
4. 未確認
5. 覚えた

### 見せ方の注意

ユーザーには「復習スケジュールを自動で調整しています」くらいに見せる。
細かい間隔やアルゴリズム名は出さない。

## 10. 初期サンプルフォルダ案

### 方針

主役はユーザーが自由に作るフォルダ。

ただし、空の状態だと使い方が伝わりにくいため、例としてサンプルフォルダを用意する。

### Phase 1 で入れるサンプルフォルダ

最初から大量に入れすぎない。

まずは「韓国旅行で使う単語」フォルダをサンプルとして入れるのがおすすめ。

理由:

- 期限設定と相性が良い
- 使用場面がわかりやすい
- ウィジェットに出たときの価値が伝わりやすい
- ユーザーが自分のフォルダを作る参考になる

### 追加サンプル候補

必要であれば、次のようなサンプルフォルダを後から追加する。

1. 海外旅行の空港・ホテル・レストラン
2. 英語の会議でよく使う言葉
3. デザイナーが覚えておくと便利な英語
4. カフェ・買い物・道案内

### サンプルフォルダの語数

初期は10-20語を推奨。

理由:

- 少なすぎるとすぐ見飽きる
- 多すぎると「自分で作る」体験よりサンプルが目立つ
- 10-20語なら使い方の見本としてちょうどよい

### フォルダの見せ方

自由登録できることを主役にしつつ、例フォルダは使い方の参考として見せる。

例:

- 韓国旅行で使う単語
- 英語会議
- デザイナー英語

「単語帳1」「単語リスト」のような名前は避ける。

## 11. 単語登録の扱い

### 方針

ユーザーが自由に単語を登録できることを主役にする。

ただし、入力項目が多すぎると面倒に感じるため、最初は最低限で登録できるようにする。

### できること

- フォルダを作る
- 単語を追加する
- 読み方を入れる
- 意味を入れる
- 例文を入れる
- 期限を設定する
- フォルダに追加する

### 見せ方

ホームとフォルダ一覧の両方から登録できるようにする。

初回は「まず3語だけ登録してみる」くらいの軽さにする。

- AI自動生成
- 音声再生
- ランキング

- アカウント必須化

## 15. 現時点のおすすめ結論

Phase 1 は「自由にフォルダを作って単語を登録する」を主役にする。

そのうえで、使用例として「韓国旅行で使う単語」フォルダを用意する。

理由:

- ユーザーの使い方を限定しすぎない
- 旅行前という例は期限設定との相性が良い
- サンプルがあると登録項目の意味が伝わりやすい
- App Store では具体例として見せやすい

まずはフォルダ作成、期限設定、単語登録、ウィジェット表示の流れを固める。

その使用例として「韓国旅行で使う単語」フォルダを画面やスクリーンショットに入れる。
